# Бенчмарк: на пальцах

## Что делают скрипты

1. **`build_eval_from_points.py`** — для каждой строки датасета (тикер + время + опционально статьи) вызывает выбранные модели, пишет одну JSONL-строку на **(точка × модель)** с предсказанием и **forward log-return** из yfinance.
2. **`run_model_benchmark.py`** — по этому JSONL считает таблицы: **Spearman** между сигналом и доходностью, **hit rate** по знаку, бутстреп-интервал и **p-value** (перестановки).

Конфиг из **`tradenews/config.env`** подхватывается скриптами автоматически (`config_env.apply_default_tradenews_env`), либо через `scripts/with_tradenews_config_env.sh`.

## Метрики простыми словами

- **Spearman (IC):** если отсортировать дни по силе **вашего** сигнала и отдельно по **фактической** доходности, насколько порядки похожи. Не требует линейной формулы «на 0.01 bias → столько-то процентов».
- **Hit rate (signed):** доля случаев, когда знак сигнала совпал со знаком доходности (грубее, чем Spearman).
- **Доверительный интервал / p:** при **маленьком N** широкие интервалы и большие p — нормально; «победитель» на одной выборке может не повториться на следующей.

## Проблемы выборки

- **Мало точек** — высокий Spearman может быть удачей; опирайтесь на CI и повтор прогона.
- **Один рынок / период** — выводы не переносятся автоматически на NYSE vs LSE или на другой год.
- **Новости ≠ вся цена** — сильный тренд или техника могут перебить новостной слой; бенч измеряет **только** вклад новостного предиктора в ранжировании.
